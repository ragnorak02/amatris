# Import Assets

Import 3D/2D assets from a source directory into the shared asset library.

## Usage
`/import-assets <source_directory>`

The source directory should contain the model/asset files (FBX, GLB, OBJ, PNG, etc.) you want to import.

## Instructions

When the user runs this command with a directory path as `$ARGUMENTS`:

1. **Validate the source directory** ã confirm the path exists and list its contents.

2. **Ask the user** (if not obvious from context):
   - **Asset name** ã suggest based on folder name, let user override (use snake_case)
   - **Dimension** ã `3d` or `2d`
   - **Category** ã one of: `characters`, `props`, `environment`, `buildings`, `foliage`, `creatures`, `npcs` (for 3d) or `characters`, `props`, `tiles`, `ui` (for 2d)
   - **Theme** ã `fantasy`, `sci-fi`, `modern`, `cartoon`, `generic`
   - **Type** ã `model`, `sprite`, `texture`, `tileset`, `audio`, `ui`
   - **Source tool** ã default `ComfyUI + Hunyuan3D-2` for 3D, ask if different
   - **Origin** ã `ai-generated` (default), `hand-made`, `third-party`

3. **Copy files** into `shared_assets/<dimension>/<category>/<asset_name>/`
   - Copy all relevant asset files (fbx, glb, gltf, obj, mtl, blend, png, jpg, etc.)
   - Skip non-asset files (thumbs.db, .DS_Store, etc.)

4. **Generate `asset.meta.json`** in the asset folder with this schema:
   ```json
   {
     "schemaVersion": 1,
     "id": "<asset_name>",
     "name": "<Display Name>",
     "type": "<model|sprite|texture|tileset|audio|ui>",
     "dimension": "<2d|3d>",
     "theme": "<fantasy|sci-fi|modern|cartoon|generic>",
     "category": "<category>",
     "tags": ["<auto-generated from name and category>"],
     "files": [
       { "name": "filename.fbx", "format": "fbx", "sizeBytes": 12345 }
     ],
     "source": {
       "tool": "<tool used>",
       "origin": "<ai-generated|hand-made|third-party>",
       "license": "internal"
     },
     "status": "raw",
     "version": 1,
     "contentHash": "sha256:<hash of primary file>",
     "created": "<ISO8601>",
     "updated": "<ISO8601>"
   }
   ```

5. **Rebuild the catalog** by running: `node scripts/asset_sync.js --rebuild-catalog`

6. **Report** what was imported ã show file count, total size, and the asset ID.

7. **Remind the user** to set `"status": "approved"` in the asset.meta.json when they're ready for the asset to sync to game repos.

## Notes
- The `contentHash` should be SHA-256 of the primary file (first FBX/GLB/PNG found)
- Asset status starts as `"raw"` ã must be manually promoted to `"approved"` before sync distributes it
- Use `node scripts/asset_sync.js --status` to check which games would receive the asset
- Use `node scripts/asset_sync.js --execute` to push approved assets to opted-in games
